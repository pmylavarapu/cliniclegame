# Clinicle

Semantle for medical diagnoses. A new secret diagnosis every day. The closer
your guess is (in semantic space) to the day's target, the higher your score.

## Stack

- Next.js 15 (App Router) + React 19 + TypeScript, static export
- Tailwind for styling
- No backend. All puzzle data is precomputed at build time.
- Data pipeline in Python: SapBERT embeddings + a deterministic date scheduler
  + LLM-generated flavor prompts.

## Directories

```
app/                    Next.js routes (Today, Archive)
components/             PuzzleGame, PuzzleLoader
lib/                    types, share string, localStorage, score decoding
public/
  vocab.json            shared 50k-word guess vocabulary
  puzzles/YYYY-MM-DD.json   one file per day
  index.json            list of available dates + latest
scripts/                Python data pipeline
data/
  diagnoses.txt         source list of single-word diagnoses (curated)
  vocab.txt             derived guess vocabulary
  schedule.json         date -> diagnosis (deterministic shuffle)
  prompts.json          diagnosis -> flavor prompt (LLM-generated)
  embeddings/           SapBERT vectors (gitignored, ~150MB)
  hf-cache/             local SapBERT weights (gitignored, ~440MB)
```

## Puzzle JSON schema

```json
{
  "date": "2026-07-22",
  "num": 203,
  "prompt": "A soft, bluish, painless swelling on the inside of a lower lip...",
  "secret": "mucocele",
  "top1000": [["mucocele", 100.0], ["mucocyst", 83.5], ...],
  "scores": "<base64 Uint16>"
}
```

`scores` is a base64-encoded Uint16 array, one entry per word in
`public/vocab.json`. Decode: `similarity_percent = stored / 100 - 30`.

## Local development

```
npm install
npm run dev   # http://localhost:3001
```

The app fetches `/index.json` on load and picks today's puzzle (or falls back
to the most recent available date).

## Data pipeline

```bash
pip install -r scripts/requirements.txt
export GOOGLE_API_KEY=...           # https://aistudio.google.com/app/apikey

# One-off (or when vocab source changes) — downloads MeSH XML (~350MB) once
python3 scripts/build_vocab.py       # data/vocab.txt
python3 scripts/schedule.py          # data/schedule.json (deterministic)

# Every time vocab changes — cached, only new phrases hit the API
python3 scripts/embed.py             # data/embeddings/*

# Add prompts (needs ANTHROPIC_API_KEY)
python3 scripts/prompts.py

# Emit puzzle JSONs for a date range (skips weakly-embedded targets)
DATE_START=2026-07-22 DATE_END=2026-08-20 python3 scripts/precompute.py
```

The GitHub Action at `.github/workflows/clinicle-daily.yml` runs this nightly
to extend the puzzle horizon and commits the generated JSON back to `main`.

## Design choices

- **Gemini semantic embeddings** (`gemini-embedding-001`, task type
  `SEMANTIC_SIMILARITY`, 768 dim). Cosine similarity between L2-normalized
  vectors drives every score in the game. General-purpose modern embeddings
  handle rare terms (loanwords, eponyms) and multi-word phrases much better
  than domain-specialized subword encoders.
- **Vocab** is `MeSH descriptors + entry terms ∪ diagnoses.txt ∪ adjuncts.txt
  ∪ top-15k common English`, capped at 80k. Multi-word phrases like
  `myocardial infarction`, `chest pain`, and `pericardial effusion` are
  first-class guesses. Anatomy, symptoms, qualifiers, and everyday
  descriptors are in the pool.
- **Quality gate**: puzzle targets whose top-neighbor cosine is below
  `MIN_TOP_SIM` (default 0.55) are skipped as poorly embedded — this keeps
  degenerate puzzles (e.g. rare loanwords the model doesn't understand) out
  of the daily rotation.
- **Scheduling** is a seeded shuffle over the deduped diagnosis list, with
  the pool refilled each time it's exhausted (each date across the horizon is
  distinct, and no diagnosis repeats within a single pass through the list).
- **Prompts** are generated by an LLM at build time from a system prompt that
  forbids naming the diagnosis or an unambiguous eponym, and are committed to
  the repo so runtime never depends on an API.
