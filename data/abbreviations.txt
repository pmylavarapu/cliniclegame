# Common medical abbreviations. Format: abbr|full expansion
# The expansion is what actually gets sent to the embedding model, so the
# abbreviation clusters near its true meaning. The user still types the
# short form (which becomes the stored vocab entry).
#
# Curated for unambiguous, widely-recognized short forms. Where an
# abbreviation has multiple common expansions, we pick the diagnosis-flavored
# one (e.g. `pd` = parkinson disease, not peritoneal dialysis; `as` =
# ankylosing spondylitis, not aortic stenosis).
